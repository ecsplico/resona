Transcriber Protocol

resona_engine_core/protocol.py
from typing import Protocol, runtime_checkable
import numpy as np

@runtime_checkable
class Transcriber(Protocol):
    """All backends implement this interface."""

    def transcribe(
        self,
        audio: np.ndarray,
        *,
        language: str = "de",
        task: str = "transcribe",
        initial_prompt: str | None = None,
        word_timestamps: bool = False,
        vad_filter: bool = False,
    ) -> TranscriptionResult:
        ...

class TranscriptionResult(TypedDict):
    text: str
    language: str
    segments: list[dict]

What this gives us

  • @runtime_checkable — can verify at startup that a backend is valid
  • Keyword-only args after audio — backends ignore what they don't support
  • Typed return — consumers know exactly what they get
  • No base class inheritance needed — duck typing with safety

Constraints

  • Backends must accept **kwargs for forward-compat with new params
  • segments format varies by backend — kept as list[dict] for flexibility

Backend Discovery via Entry Points

How backends register themselves
Backend pyproject.toml
# packages/engine-faster-whisper/pyproject.toml

[project.entry-points."resona.backends"]
faster-whisper = "resona_engine_faster_whisper.transcriber:FastWhisperTranscriber"
Registry discovers at startup
# resona_engine_core/registry.py

from importlib.metadata import entry_points

def get_transcriber(
    backend: str | None = None,
) -> Transcriber:
    name = backend or config("RESONA_BACKEND")
    eps = entry_points(group="resona.backends")
    for ep in eps:
        if ep.name == name:
            cls = ep.load()
            instance = cls(device=_detect_device())
            assert isinstance(instance, Transcriber)
            return instance
    raise ValueError(
        f"Backend '{name}' not found. "
        f"Installed: {[e.name for e in eps]}"
    )

Why entry points?

  • Zero coupling — core never imports backend code directly
  • Install = register — uv pip install resona-engine-voxtral automatically makes "voxtral" available
  • One backend per image — Docker image installs one backend package, entry point resolves it
  • CLI auto-start — resona batch --backend voxtral knows to uv run resona-engine-voxtral

Singleton Lifecycle (unchanged pattern)

One transcriber per process, lazy-loaded
# resona_engine_core/registry.py

_transcriber: Transcriber | None = None
_lock = threading.Lock()

def get_transcriber(backend: str | None = None) -> Transcriber:
    global _transcriber
    if _transcriber is None:
        with _lock:
            if _transcriber is None:
                _transcriber = _load_from_entrypoint(backend)
    return _transcriber
