Backend Resolution Flow

What happens when you run: resona batch ./audio/ --backend voxtral
1. Try configured remote backends
   │  GET http://gpu-server:7000/health
   │
   ├─ reachable → use remote server (--backend flag ignored for remote)
   │
   └─ unreachable → try auto-start (SSH tunnel / docker compose)
      │
      ├─ auto-start worked → use remote server
      │
      └─ still unreachable → local fallback
         │
2. Resolve backend name
         │  --backend flag  →  "voxtral"
         │  OR config.json  →  "faster-whisper"
         │  OR hardcoded     →  "faster-whisper"
         │
3. Map backend name to package
         │  "voxtral"  →  resona-engine-voxtral
         │  "faster-whisper"  →  resona-engine-faster-whisper
         │
4. Spawn local engine
         │  uv run resona-engine-voxtral
         │  poll /health until ready
         │
5. Transcribe + postprocess
         │  POST /transcribe → raw text
         │  PostprocessPipeline (from postprocess.json) → md

LocalEngine Changes

resona_cli/local_engine.py — updated
class LocalEngine:
    def __init__(
        self,
        backend: str = "faster-whisper",
        model: str | None = None,
        timeout: float = 120.0,
    ):
        self.backend = backend
        self.model = model
        self.timeout = timeout
        # package name derived from backend name
        self._package = f"resona-engine-{backend}"

    def __enter__(self):
        # spawn: uv run resona-engine-{backend}
        self._process = subprocess.Popen(
            ["uv", "run", self._package],
            env={**os.environ, "PORT": str(self._port)},
            ...
        )
        self._wait_for_health()
        return self

Each Backend Package is Runnable

packages/engine-voxtral/pyproject.toml
[project]
name = "resona-engine-voxtral"
dependencies = [
    "resona-engine-core",
    "transformers",
    "torch",
]

# Register as a backend
[project.entry-points."resona.backends"]
voxtral = "resona_engine_voxtral.transcriber:VoxtralTranscriber"

# Make it runnable: uv run resona-engine-voxtral
[project.scripts]
resona-engine-voxtral = "resona_engine_core.app:main"

The [project.scripts] entry point means uv run resona-engine-voxtral launches the core FastAPI app. On startup, the app calls get_transcriber() which discovers the voxtral backend via entry points.

Summary of All Config Sources

~/.resona/ full layout
~/.resona/
├── config.json          ← remote backends + default_backend
├── replacements.json    ← static replacement rules (local mode)
└── postprocess.json     ← full pipeline config: replacements + LLM steps

postprocess.json references replacements.json as a source. If postprocess.json doesn't exist, CLI falls back to just replacements.json for backward compat.
