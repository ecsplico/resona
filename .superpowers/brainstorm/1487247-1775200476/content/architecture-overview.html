Key Design Decisions

Transcriber Protocol

Formal typing.Protocol in ws-engine-core. All backends implement it. Discovered via Python entry points.

One Backend Per Image

Each backend is a separate package + Dockerfile. No bloated images. ASR_MODE still selects at runtime within a package.

Postprocessing Library

ws-postprocess used by ws-api (from DB) and ws-cli (from local JSON file). Engine stays pure ASR.

Local Fallback

CLI reads ~/.whisper-server/config.json for default backend + replacements. --backend flag overrides.

Data Flow

Server path (ws-api → engine)
Client ──POST /jobs──▶ ws-api
                        │ saves file, creates PENDING job
                        │
                   TranscribeTask
                        │ fetches replacements + prompt from DB
                        │ POSTs audio + initial_prompt to engine
                        ▼
                   ws-engine-core + backend
                        │ transcribes audio → {text, language, segments}
                        │ (NO replacements applied here anymore)
                        ▼
                   ws-api
                        │ applies ws-postprocess.apply_replacements()
                        │ stores text + md + segments
                        ▼
Client ──GET /job/{id}──▶ COMPLETED with transcript
Local path (ws-cli direct)
ws-cli batch ──▶ no server found
                  │ reads ~/.whisper-server/config.json
                  │   → default_backend, replacements
                  │
                  │ spawns uv run ws-engine-faster-whisper
                  │ POSTs audio to local engine
                  ▼
             ws-engine-core + backend
                  │ returns {text, language, segments}
                  ▼
             ws-cli applies ws-postprocess.apply_replacements()
                  │ using local replacements.json
                  ▼
             writes .txt / .md to output dir
